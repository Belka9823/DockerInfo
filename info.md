# Практическое руководство и заметки о Docker

Этот файл содержит основные команды, структуры конфигурационных файлов и шпаргалки для ежедневной работы с контейнерами.

---

## 1. Docker Hub и работа с образами
**Docker Hub** — центральный репозиторий образов. 

*   `docker pull <image>:<tag>` — скачать образ (например, `postgres:15`).
*   `docker images` — список всех скачанных образов.
*   `docker rmi <id>` — удалить ненужный образ.
*   `docker search <name>` — поиск образов в хабе через терминал.

---

## 2. Dockerfile: Шпаргалка по командам
`Dockerfile` — это пошаговая инструкция для сборки собственного образа.

| Инструкция | Описание | Пример |
| :--- | :--- | :--- |
| **FROM** | Базовый образ | `FROM python:3.9-slim` |
| **WORKDIR** | Рабочая директория внутри контейнера | `WORKDIR /app` |
| **COPY** | Копирование файлов с хоста в контейнер | `COPY . .` |
| **RUN** | Выполнение команд при сборке | `RUN pip install -r requirements.txt` |
| **ENV** | Переменные окружения | `ENV PORT=8080` |
| **EXPOSE** | Документирование порта | `EXPOSE 8080` |
| **CMD** | Команда при запуске контейнера | `CMD ["python", "app.py"]` |

**Сборка образа:** `docker build -t my-app:v1 .`

---

## 3. Управление контейнерами (Основные команды)
*   `docker run -d -p 80:80 --name web nginx` — запустить контейнер в фоновом режиме (`-d`) с пробросом порта 80 на 80.
*   `docker ps` — список запущенных контейнеров.
*   `docker ps -a` — список всех контейнеров (включая остановленные).
*   `docker stop <id/name>` — остановить контейнер.
*   `docker rm -f <id/name>` — принудительно удалить контейнер.
*   `docker exec -it <name> bash` — зайти внутрь работающего контейнера.

---

## 4. Docker Compose
Инструмент для запуска нескольких сервисов (например, БД + Сайт). Все описывается в файле `docker-compose.yml`.

**Пример структуры:**
```yaml
version: '3.8'
services:
  web:
    build: .
    ports:
      - "5000:5000"
  db:
    image: postgres:15
    environment:
      POSTGRES_PASSWORD: example
